# 0.2.4 UI Restructure Implementation Plan

## Feature Request Summary

Requested UI changes for `0.2.4`:

- replace the current stacked main-area organization with a fixed 2-column layout
- make the left column a navigation column with:
  - a top division for `Projects`, `Favorites`, `Files`, and `Notes`
  - `Favorites` sub-tabs for `Project Favorites` and `Global Favorites`
  - `Files` sub-tabs for `Checked Out Files` and `Reference Files`
  - a bottom `Directories` division with `Source` and `Local` tabs
- make the right column the expanded `Files` workspace
- move the current source-folder display into the right column
- convert the extension filter from an always-expanded section into a button-driven dialog with an adjacent preview label
- keep the file search on the same row, aligned right
- move `Directory > Controlled Files` and `Directory > Notes` out of the main layout and into a separate popup reachable from the Files menu
- remove the dormant `Global Notes` feature entirely

## Current State Review

- `document_control/mixins/ui.py` currently builds the Main tab as vertically stacked collapsible `Projects` and `Files` sections.
- The current `Projects` section uses a 3-way horizontal splitter for tracked projects, favorites/local-file lists, and notes.
- The current `Files` section uses a `Source` / `Local` tab layout, with the source view also containing the directory-level controlled-files and file-notes tables.
- Extension filtering is always visible inside the right-side source file workspace.
- Existing behavior and tests already depend on the current widgets and helper methods, especially:
  - `favorites_tabs`
  - `files_scope_tabs`
  - source/local directory widgets
  - checked-out/reference actions routed from the active list tab

## Design Decisions

### 1. Main Layout

Adopt a fixed horizontal splitter for the Main tab:

- left column:
  - top: primary navigation tabs
  - bottom: directories group
- right column:
  - expanded files workspace

This matches the request more closely than the existing collapsible vertical stack and keeps the screen real estate predictable.

### 2. Left-Column Navigation

Use one top-level `QTabWidget` for:

- `Projects`
- `Favorites`
- `Files`
- `Notes`

Recommended contents:

- `Projects`
  - current project label
  - tracked-project search
  - tracked-project list
- `Favorites`
  - nested tabs:
    - `Project Favorites`
    - `Global Favorites`
- `Files`
  - nested tabs:
    - `Checked Out Files`
    - `Reference Files`
- `Notes`
  - project notes list and actions only

### 3. Directories Area

Keep the lower-left `Directories` section, but limit it to navigation only:

- `Source`
  - tracked source directories
  - source directory browser
- `Local`
  - tracked local directories
  - local directory browser

This means the lower-left area should no longer duplicate file lists.

### 4. Right-Side Files Workspace

The right column should become the main file browser workspace for the currently selected scope:

- top row:
  - current folder label
  - `Extension Filter` button
  - compact filter-preview label
  - file search box aligned to the right
- center:
  - main file table
- no always-visible directory metadata tables in this pane

Recommended implementation detail:

- keep separate source/local file tables internally if that reduces risk
- switch the visible workspace based on the active `Directories` tab
- preserve existing methods and widget names where practical to reduce regression risk

### 5. Extension Filter Dialog

Replace the current inline extension filter `QGroupBox` with a dialog launched from the files toolbar row.

Recommended dialog contents:

- filter mode selector
- extension list editor
- preset access
- add/remove/clear controls

Recommended preview label behavior:

- show `No Filter` when inactive
- show `Include Only: .dwg, .xls` or `Exclude: .bak, .tmp` when active
- truncate visually in the label if needed, but keep the full value in a tooltip

### 6. Directory Details Popup

Move `Controlled Files` and `Directory Notes` into a separate popup window reachable from the Files menu.

Recommended behavior:

- one reusable dialog/window instance
- tabs:
  - `Controlled Files`
  - `File Notes`
- source-directory context follows the currently selected source directory
- existing actions stay available:
  - refresh
  - force check in
  - view file notes

Recommended reason:

- keeps the main layout focused on navigation + file browsing
- avoids deleting working functionality while satisfying the request to reclaim space

### 7. Global Notes Removal

Remove the dormant `Global Notes` feature rather than carrying it into the new layout.

Recommended scope:

- remove the `Global Notes` UI
- remove startup/shutdown load-save wiring for `global_notes`
- remove persistence support for `global_notes.json`
- remove tests that only exist for the unused feature

Reason:

- the feature is not part of the current product direction
- preserving it would add unnecessary UI and lifecycle complexity during the restructure
- file notes and project notes remain separate and should be preserved

## Compatibility Strategy

This refactor should stay conservative around existing behavior:

- preserve existing widget-backed workflows where possible rather than rewriting the action layer at the same time
- prefer compatibility shims if active-tab routing depends on old tab structures
- avoid changing persistence formats for a layout-only feature
- except for the intentional removal of the unused `Global Notes` persistence path

This is especially important for:

- favorites/reference action dispatch
- source/local file refresh behavior
- context-menu wiring
- tests that rely on existing widget names or tab indexes
- file-note behavior, which must remain intact while `Global Notes` is removed

## Recommended Build Phases

### Phase 1: Layout Refactor

Goal:

- land the 2-column main layout without changing core file operations

Tasks:

- rebuild the Main tab container in `document_control/mixins/ui.py`
- move project/favorites/files/notes into left-column tabs
- move directories into the lower-left group
- keep the right column focused on file browsing

Success criteria:

- all existing main workflows remain reachable
- no loss of tracked-directory, project, favorites, reference, or note functionality

### Phase 2: Extension Filter Dialog

Goal:

- replace the inline filter block with a button + preview pattern

Tasks:

- extract the existing filter controls into a dialog builder/helper
- add inline preview label generation
- keep presets and existing filter persistence behavior unchanged

Success criteria:

- filter editing still works
- file refresh still respects the selected mode/extensions
- preview text accurately reflects active settings

### Phase 3: Directory Details Popup

Goal:

- remove directory metadata tables from the main layout without losing those workflows

Tasks:

- create a popup/dialog for directory-level `Controlled Files` and `File Notes`
- add a Files menu action to open it
- reuse existing table refresh and context-menu logic

Success criteria:

- users can still inspect and act on controlled files and directory notes
- the popup reflects the current directory context correctly

### Phase 4: Global Notes Removal

Goal:

- remove the unused global-notes feature cleanly while preserving project notes and file notes

Tasks:

- remove `Global Notes` UI builders and window wiring
- remove `global_notes` startup/shutdown load-save calls
- remove any now-unused persistence constants/helpers for `global_notes.json`
- remove obsolete global-notes tests and doc references

Success criteria:

- no `Global Notes` UI remains
- startup/shutdown does not depend on `global_notes.json`
- project notes and file notes still behave the same

### Phase 5: Wiring, Cleanup, and Validation

Goal:

- stabilize the layout and eliminate routing regressions

Tasks:

- verify active-tab action routing for favorites, checked-out files, and references
- verify source/local switching after the file workspace moves
- tighten sizing, labels, and menu wording to match the mockups
- update user-facing docs after implementation lands

Success criteria:

- navigation feels coherent
- no action silently targets the wrong list or scope

## Open Questions Requiring Confirmation

### 1. Popup Type For Directory Details

The request says "pop-up window." That could mean:

- modal dialog
- modeless dialog
- separate utility window

Recommendation:

- use a reusable modeless dialog so users can inspect directory details without blocking the main workspace

### 2. Right-Pane Local Files Presentation

The request explicitly describes the right pane as the expanded `Files` section and references the current source folder label, but does not separately describe how Local file browsing should appear there.

Recommendation:

- keep source and local file tables as the right-pane workspace selected by the lower-left `Directories` tab, so the layout remains consistent for both scopes

## Recommendation

This should be implemented as a layout and workflow-routing refactor, not as a data-model change.

The safest path is:

1. rebuild the layout conservatively in `ui.py`
2. preserve existing widget contracts where possible
3. add small compatibility shims for old tab assumptions
4. move directory-only detail tables into a dedicated popup instead of deleting that functionality
5. remove `Global Notes` deliberately as part of the same planned cleanup
