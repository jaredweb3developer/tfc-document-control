# `0.2.2` Files Tabs + Local Transfer Scope

## Purpose

Capture the functional scope implemented for `0.2.2` after the `0.2.1` baseline.

## Agreed Scope

1. Rename the Main-page `Source Files` section to a generic `Files` section.
2. Add `Source` and `Local` tabs inside the Files section.
3. Add Local-tab support for tracking and browsing local directories without requiring OS-native file explorer navigation.
4. Allow selecting local files in the Local tab and copying them into source via `Add Local File(s) To Source`.
5. Destination selection for local-to-source transfer must default to the most recently loaded source folder but allow user override.

## Implemented Behavior

- `Files` section now contains:
  - `Source` tab: existing source workflow (tracked source roots, source tree, source files table, controlled files and notes).
  - `Local` tab: tracked local roots, local tree browser, and local files table with Explorer-style metadata columns.
- Local tab includes context/menu actions to:
  - open selected local files
  - refresh local file list
  - view current local folder location
  - add selected local files to source
- `Add Local File(s) To Source` opens a destination chooser dialog:
  - defaults to the last source folder loaded in Source tab
  - pre-populates tracked source roots
  - allows browsing to another existing destination folder

## Persistence Additions

Project config (`dctl.json`) now stores:

- `local_directories`: list of tracked local roots per project
- `selected_local_directory`: currently selected local root per project

These fields are additive and do not replace existing `sources` / `selected_source` fields.
